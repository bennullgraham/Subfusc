Stack

This site is cached by varnish. Behind varnish, nginx either serves static files directly or hands off application requests to gevent. gevent serves a lightweight Python application written in the Flask web framework.

Flask-Assets is employed to bundle and filter CSS and Javascript source files. Javascript is run through rJsmin, and CSS through pyScss, a Python implementation of the compiler for the SCSS variant of Sass. All Javascript source is loaded asynchronously with a variant of $script.js, with the exception of one or two small inline snippets (one of which is, obviously, the asynchronous loader itself).

Performance

Ignoring transport-layer delays, the stack employed allows thousands of requests per second with response times in the tens of milliseconds. These figures were obtained with: 

$ sudo httperf --hog --server=bgraham.com.au --uri=/ --num-conns=5000 --rate=2500 --num-calls=3

While the server may respond incredibly quickly to requests, the main delay is in the network round-trip time - i.e., the time it takes for a request to physically traverse the Internet. The order and loading of resources has been optimised to lower these delays to the absolute minimum. The chart alongside shows the network timeline for a request made in Melbourne, Australia to the server hosting this site - an EC2 instance in Tokyo, Japan. The blue vertical line represents the DOMContentLoaded event, which, roughly speaking, is when the page begins to display. The bare minimum of resources are requested before allowing this event to occur. Other resources are then requested, and the page is 'finished' at the red vertical line.
